# Output Analysis Guide

## Output File Inventory

| Extension | Name | Description | Generated By |
|-----------|------|-------------|--------------|
| `.err` | Error file | Errors, warnings, simulation status | Always |
| `.eso` | Standard output | Raw time-series data (binary) | Always |
| `.csv` | CSV output | Converted time-series data | `--readvars` flag |
| `.mtr` | Meter output | Raw meter data (binary) | Always |
| `.html`/`.htm` | HTML report | Summary tables and reports | `Output:Table:SummaryReports` |
| `.sql` | SQLite database | Complete structured output | `Output:SQLite` |
| `.eio` | Info output | EnergyPlus initialization info | Always |
| `.rdd` | Report Data Dict | Available output variables | Always |
| `.mdd` | Meter Data Dict | Available meters | Always |
| `.bnd` | Branch/Node | HVAC system topology | Always |
| `.shd` | Shading | Shading calculation details | Always |
| `.dxf` | Drawing | 3D geometry (if requested) | `Output:Surfaces:Drawing` |

## .err File Structure

### Error Classification
- **Fatal**: Simulation cannot continue. Must fix before re-running.
- **Severe**: Simulation completed but results may be unreliable.
- **Warning**: Minor issues. Results are generally valid.

### Format
```
** Fatal  ** Error message text
**   ~~~   ** Continuation of error details
** Severe  ** Error message text
**   ~~~   ** Additional information
** Warning ** Warning message text
```

### Reading Strategy
1. Check Fatal errors first (at bottom of file)
2. Review all Severe errors
3. Scan Warnings for patterns

### Error Summary Line
Near the end: `EnergyPlus Completed Successfully-- N Warning; M Severe Errors; Simulation Status`

## .csv File Format

### Structure
```
Date/Time,Variable1 [Unit](Frequency),Variable2 [Unit](Frequency),...
01/01  01:00:00,21.5,150.3,...
01/01  02:00:00,21.3,148.7,...
```

### Column Header Format
```
Zone:Variable Name [Unit](Frequency)
```
Example: `THERMAL ZONE: SPACE 101:Zone Mean Air Temperature [C](Hourly)`

### Date/Time Format
```
MM/DD  HH:MM:SS
```
Example: `01/15  14:00:00`

## SQLite Database Schema

### Key Tables

#### `ReportDataDictionary`
Maps variable indices to names.
```sql
SELECT ReportDataDictionaryIndex, KeyValue, Name, Units
FROM ReportDataDictionary
WHERE Name LIKE '%Temperature%';
```

#### `ReportData`
Time-series values for all output variables.
```sql
SELECT rd.Value, t.Month, t.Day, t.Hour
FROM ReportData rd
JOIN Time t ON rd.TimeIndex = t.TimeIndex
WHERE rd.ReportDataDictionaryIndex = 42
ORDER BY t.TimeIndex;
```

#### `Time`
Timestamp reference table.
```sql
SELECT TimeIndex, Year, Month, Day, Hour, Minute, Interval
FROM Time;
```

#### `TabularDataWithStrings`
All summary report tables stored in normalized form.
```sql
SELECT ReportName, TableName, RowName, ColumnName, Value, Units
FROM TabularDataWithStrings
WHERE TableName = 'End Uses';
```

#### `Zones`
Zone definitions.
```sql
SELECT ZoneIndex, ZoneName, FloorArea, Volume
FROM Zones;
```

### Common SQL Queries

**Site Energy Summary:**
```sql
SELECT RowName, Value, Units
FROM TabularDataWithStrings
WHERE TableName='Site and Source Energy'
AND ReportName='AnnualBuildingUtilityPerformanceSummary';
```

**End Uses by Fuel:**
```sql
SELECT RowName, ColumnName, Value, Units
FROM TabularDataWithStrings
WHERE TableName='End Uses'
AND ReportName='AnnualBuildingUtilityPerformanceSummary'
AND Value != '0.00';
```

**Unmet Hours:**
```sql
SELECT RowName, Value, Units
FROM TabularDataWithStrings
WHERE TableName='Comfort and Setpoint Not Met Summary'
AND ReportName='AnnualBuildingUtilityPerformanceSummary';
```

**Building Area:**
```sql
SELECT RowName, Value, Units
FROM TabularDataWithStrings
WHERE TableName='Building Area'
AND ReportName='AnnualBuildingUtilityPerformanceSummary';
```

**Zone Component Loads (Peak Cooling):**
```sql
SELECT RowName, ColumnName, Value, Units
FROM TabularDataWithStrings
WHERE TableName='Estimated Cooling Peak Load Components'
AND ReportName='ZoneComponentLoadSummary';
```

**Monthly Energy by End Use:**
```sql
SELECT RowName, ColumnName, Value
FROM TabularDataWithStrings
WHERE TableName='End Uses By Month'
AND ReportName='AnnualBuildingUtilityPerformanceSummary';
```

**All Available Report Tables:**
```sql
SELECT DISTINCT ReportName, TableName
FROM TabularDataWithStrings
ORDER BY ReportName, TableName;
```

## HTML Report Tables

### Annual Building Utility Performance Summary
Key tables within this report:
- **Site and Source Energy**: Total site energy (GJ), source energy, net site energy
- **End Uses**: Energy by end-use category (Heating, Cooling, Lighting, Equipment, etc.) and fuel type
- **Building Area**: Total, conditioned, and unconditioned floor area (m2)
- **Utility Use Per Conditioned Floor Area**: Energy intensity (MJ/m2)

### Input Verification and Results Summary
- Zone summary (area, volume, multiplier)
- Envelope summary (wall area, window area, WWR)
- HVAC sizing results

### Comfort and Setpoint Not Met Summary
- Time heating setpoint not met (hours)
- Time cooling setpoint not met (hours)
- Time heating setpoint not met during occupied (hours)
- Time cooling setpoint not met during occupied (hours)

## Key Energy Metrics

| Metric | Source | Unit | Significance |
|--------|--------|------|-------------|
| Total Site Energy | Site and Source Energy table | GJ | Total energy consumed at building |
| Energy Use Intensity (EUI) | Utility Use Per Conditioned Floor Area | MJ/m2 or kWh/m2 | Benchmark comparison metric |
| Peak Heating Load | Zone Component Load Summary | W | HVAC sizing reference |
| Peak Cooling Load | Zone Component Load Summary | W | HVAC sizing reference |
| Unmet Heating Hours | Comfort table | hours | HVAC performance indicator |
| Unmet Cooling Hours | Comfort table | hours | HVAC performance indicator |

## .rdd and .mdd Files

### .rdd (Report Data Dictionary)
Lists all available output variables. Format:
```
Output:Variable,*,Zone Mean Air Temperature,hourly; !- Zone Average [C]
Output:Variable,*,Zone Ideal Loads Supply Air Total Heating Energy,hourly; !- Zone Sum [J]
```

Use to discover what variables can be requested via `Output:Variable` in the IDF.

### .mdd (Meter Data Dictionary)
Lists all available meters. Format:
```
Output:Meter,Electricity:Facility,hourly; !- [J]
Output:Meter,NaturalGas:Facility,hourly; !- [J]
Output:Meter,Heating:EnergyTransfer,hourly; !- [J]
```

Use to discover what meters can be requested via `Output:Meter` in the IDF.
